# 提示词工程与评测（工程规范摘要）

以 `doubao-MD-Doc/进阶使用/提示词工程.md` 为准；本文件提供工程侧可执行要点。

## 选模型决定提示词风格

- 带深度思考的文本生成模型：给目标与约束即可，让模型自拟计划与校验。
- 无/关闭深度思考：适合强模板化场景；指令更明确，给精确步骤、格式与样例。

## 先评测再迭代

- 建立评测与回归机制：更新 → 测试 → 调试 → 评估；版本化管理变更与差异。
- 评测集建议覆盖：正例/负例/边界条件/多语料；规模建议几百到几千样本。

## 角色与指令优先级

- `system`：高优先级规则与业务逻辑（身份、语气、边界、工具使用规范）。
- `user`：需求与问题。
- `assistant`：输出。
- 工具相关：按 API 约定回传工具输出（不要混入 assistant 文本）。

## 用结构化内容表达“规则/示例/上下文”

- 用 Markdown 分区：Identity / Instructions / Examples / Context。
- 必要时用轻量 XML 标签显式划界示例与材料，减少模型误读。

## Few-shot 与步骤约束

- Few-shot：给少量输入/输出示例，让模型泛化到新问题。
- 指定步骤：对复杂任务给执行步骤与完成标准，保证顺序与完整性。
